## 啟動與方向

- [ ] 直式開啟時顯示「請將手機轉為橫式」，沒有可誤觸的遊戲控制。
- [ ] 轉為橫式後顯示開始畫面，按鈕文字為「開始觸控遊戲」。
- [ ] 點擊開始後不要求 Pointer Lock，也不顯示滑鼠鎖定錯誤。
- [ ] Fullscreen 或 orientation lock 成功時可持續橫式。
- [ ] Fullscreen 或 orientation lock 不支援／被拒絕時仍可在橫式開始。
- [ ] 遊玩中轉回直式會暫停世界並顯示方向閘門；任務秒數仍繼續。
- [ ] 回到橫式後顯示恢復畫面，不會自行誤觸恢復。

## 觸控駕駛

- [ ] 上、下、左、右各自移動 RBC 的局部截面位置。
- [ ] 同時按兩個方向可斜向移動。
- [ ] 手指滑出按鍵、系統中斷觸控或旋轉裝置後不會卡鍵。
- [ ] 手機固定前方視角；滑動畫面不改變 yaw／pitch。
- [ ] PAUSE 可停止世界，點擊恢復後回到原狀態。

## 氣體交換

- [ ] 進入體微血管或肺泡微血管時 O／C 觸控鍵可快速連打。
- [ ] 每次 pointerdown 只計一次，不會因長按自動灌入多次。
- [ ] 一般狀態 O、C 各 3 次可完成；CO 中毒時各 9 次可完成。
- [ ] QTE 面板、O／C 按鍵與任務倒數在橫式小螢幕不互相遮住關鍵數字。

## BP 與音量鍵

- [ ] 畫面 BP ＋按住可提高 BP／速度，低血壓停滯時仍有效。
- [ ] 畫面 BP −按住可降低 BP／速度，低血壓停滯時不妨礙 BP ＋規則。
- [ ] 按硬體音量提高／降低鍵時，記錄瀏覽器是否送出事件及 BP 是否變動。
- [ ] 若硬體鍵只改系統音量而未改 BP，標記為平台限制，不視為遊戲失敗。
- [ ] 無論硬體鍵結果如何，畫面 BP ＋／−可完成整關。

## Android Chrome

- [ ] 返回／首頁／應用切換後控制狀態清空，不會持續移動。
- [ ] 瀏覽器工具列顯示或收合時 HUD、Canvas 與觸控區重新排版。
- [ ] 全螢幕拒絕、允許各測一次；兩者都不阻止遊戲。
- [ ] Console 無專案持續性 error。

## iOS／iPadOS Safari

- [ ] 瀏海、Dynamic Island、Home Indicator 與 safe area 不遮住控制。
- [ ] Safari 工具列展開／收合時使用動態 viewport，不產生捲動或空白帶。
- [ ] 沒有一般元素 Fullscreen 或 orientation lock 時，方向閘門仍正確。
- [ ] 從背景回前景後停在可恢復狀態，控制不黏住。
- [ ] Console 無專案持續性 error。

## 效能與長時間

- [ ] 連續遊玩 60 秒，主觀操作無持續卡頓或觸控遺失。
- [ ] 手機 renderer pixel ratio／resolution scale 已套用，不以桌面高 DPI 全解析度過度渲染。
- [ ] 四個角落 HUD、觸控區、QTE、狀態與過場無裁切。
- [ ] 至少完成一關、一次 QTE、一次暫停、一次旋轉與一次重試。
